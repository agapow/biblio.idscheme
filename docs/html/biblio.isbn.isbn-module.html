ISBN handling functions.

ISBNs are commonly used in queries on library and merchant databases. However, variation in formatting

Features

  • Functions for parsing and interconverting ISBN-10 and ISBN-13.
  • Support for old-style British 9-digit SBNs.
  • A unified class for handling ISBNs.

Caveats

Formatting and whitespace is stripped before an ISBN is parsed. It would be useful to include functionality for the reverse, consistently formatting ISBNs. However the length of the internal fields can vary (while still adding up to the same length), and without extensive lists of group and publisher codes, it is impossible.

Given that all SBNs have an equivalent ISBN-10 (but not vice-versa) and all ISBN-10s have an equivalent ISBN-13 (but not vice-versa), the question of how to handle failed conversions arises. Rather than have the user guess which codes are convertible, or try conversion and catch exceptions, failed conversions return None.
